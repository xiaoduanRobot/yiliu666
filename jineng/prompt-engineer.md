# prompt-engineer - 提示词工程师

## 技能简介

提示词工程师专注于AI提示词的设计与优化，运用少样本学习、思维链、结构化输出等系统化工程技术，旨在提升大语言模型的响应效率与准确性。

## 核心关键词

提示词工程、AI提示优化、大语言模型、少样本学习、思维链、结构化输出、提示词模板

## 核心技术

### 1. 少样本学习（Few-shot Learning）

通过在提示词中提供1-3个示例，让模型学习任务模式。

```markdown
# 示例：情感分类
输入：我今天考试得了满分！  → 情感：积极
输入：手机摔坏了，好难过。  → 情感：消极
输入：今天吃了火锅，很满足。→ 情感：？
```

### 2. 思维链（Chain of Thought）

引导模型展示推理过程，提升复杂问题的准确性。

```markdown
问题：小明有5个苹果，小红给了他3个，他又吃了2个，还剩多少？
解题步骤：
1. 小明原来有5个苹果
2. 加上小红给的3个 → 5 + 3 = 8个
3. 吃掉2个 → 8 - 2 = 6个
答案：6个
```

### 3. 结构化输出

要求模型以特定格式返回结果（如JSON、XML）。

```markdown
请以JSON格式返回用户信息：
{
  "name": "姓名",
  "age": "年龄",
  "city": "城市"
}
```

### 4. 角色设定

为模型设定特定角色，增强回答的专业性和针对性。

```markdown
你是一位资深Python后端工程师，擅长Django/Flask框架。
请用专业技术术语回答以下问题。
```

### 5. 指令分解

将复杂任务拆分为简单步骤，逐个执行。

```markdown
任务：帮我写一篇产品介绍文章
步骤：
1. 确定产品核心卖点
2. 列出3个典型使用场景
3. 撰写开头段落
4. 撰写主体内容
5. 撰写结尾CTA

请从第1步开始执行。
```

## 提示词模板

### 模板1：通用问答模板

```
【角色】你是一位[领域]专家
【背景】用户遇到了[问题描述]
【要求】
- 回答要[具体要求]
- 避免[需要避免的内容]
- 语言风格：[正式/口语化]

【问题】
[用户问题]
```

### 模板2：代码描述]
【输入】[输入参数]
【输出】[输出格式]
【要求】
- 使用类型提示
- 添加异常处理
- 包含必要的注释
```

### 模板3：数据分析模板

```
【数据】[数据描述]
【目标】[分析目标]
【步骤】
1. 数据清洗
2. 探索性分析
3. 可视化
4. 结论总结

请执行分析并给出结论。
```

## 最佳实践

1. **明确性**：指令要具体清晰，避免模糊表述
2. **分隔符**：使用```、###等分隔不同部分
3. **顺序敏感**：重要信息放在提示词开头
4. **迭代优化**：根据输出效果不断调整提示词
5. **长度控制**：避免过长提示词，保持简洁

## 在小端AI中的应用

小端AI可以通过优化提示词实现：
1. **更准确的回答**：结构化输出+角色设定
2. **复杂推理**：思维链技术
3. **个性化服务**：少样本学习用户偏好
4. **任务自动化**：指令分解执行

## 注意事项

1. 提示词需要根据具体模型调整
2. 复杂的提示词不一定效果更好
3. 需要通过多次测试优化提示词
4. 结构化输出要注意JSON格式正确性
